# Current vs Target State

## Current State (As-Is)

The existing placement system exhibits two primary limitations that constrain its strategic value:

### 1. Document Extraction Fragility
The system struggles to reliably extract structured information from resumes created using graphics-heavy tools (e.g., Canva-based PDFs).  
This results in partial or missing candidate data, creating downstream blind spots in the matching process.

### 2. Matching Accuracy Ceiling
The current matching logic achieves approximately **50–60% accuracy**, driven by coarse weighting across GPA, skills, experience, and program of study.

At this level, a significant portion of placements require manual intervention, undermining the efficiency gains of automation.

---

## Target State (To-Be)

The target system state is defined by:

- Robust extraction across diverse resume formats
- Granular, transparent matching logic aligned with faculty expectations
- Reduced need for manual correction
- Compatibility with a secure, server-hosted deployment environment

Success is measured not only by accuracy improvements, but by the system’s reliability and maintainability under real administrative usage.
