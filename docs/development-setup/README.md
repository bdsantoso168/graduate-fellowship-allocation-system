# Development Environment Setup

This folder documents the **development environment and tooling decisions** for the Graduate Fellowship Allocation System consulting project.

## Purpose

The goal of this documentation is to capture **stable, reusable setup knowledge** that supports:

* Team onboarding
* Environment consistency
* Long-term maintainability
* Portfolio-level explanation of technical decision-making

This is **not** a step-by-step installation guide.
Detailed procedural setup (including trial-and-error, OS-specific issues, and credentials) is intentionally maintained outside this repository due to security and VPN constraints.

## Scope

This folder includes:

* Development tooling rationale (e.g., VS Code)
* High-level environment configuration patterns
* Conceptual database and data-layer documentation
* Key setup-related decisions and constraints

## Security & Data Constraints

All live development, real data access, and system execution occur within a **secure university VPN environment**.
As a result:

* No credentials, internal paths, or sensitive configurations are stored here
* No real student or institutional data is included
* Examples are illustrative and sanitized

## How This Folder Evolves

Content in this folder is updated **only when setup decisions stabilize**.
Ongoing experimentation and iterative notes are tracked separately in internal documentation (e.g., Excel working logs).

This separation ensures the repository remains:

* Clean
* Explainable
* Portfolio-ready
