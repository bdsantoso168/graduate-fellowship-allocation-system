# Graduate Fellowship Allocation System  
**ISOM 424 – Consulting Practicum**

This repository documents a **faculty-supervised consulting engagement** focused on improving the accuracy and efficiency of a graduate fellowship allocation system for an academic institution.

## Project Context
The existing fellowship allocation process is highly manual, requiring significant administrative effort and producing inconsistent matching outcomes. This project aims to support the transition toward a more reliable, scalable, and data-informed allocation system.

## Scope of This Repository
Due to **data sensitivity and institutional security requirements**, the live system development, real data, and testing are conducted within a secure university VPN environment.

This repository is intentionally used for:
- High-level system documentation
- Process analysis (current vs. target state)
- Sanitized meeting notes and decision logs
- Architecture and design thinking artifacts

**No real student data, credentials, or internal system files are stored here.**

## Technology Stack (Project-Level)
- Python (backend logic)
- SQL (data layer)
- Secure Linux-based hosting (university-managed)
- Internal CRM integration (restricted environment)

## Status
Active consulting engagement (14-week duration).

## Disclaimer
This repository is for **documentation and portfolio purposes only**.  
All sensitive implementation details remain within approved institutional systems.
